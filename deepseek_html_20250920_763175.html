1 • Sistema Operativo

Características del Sistema Operativo Seleccionado

Para el despliegue del sistema ERP_SENA_Final, se recomienda utilizar Ubuntu Server LTS como sistema operativo principal, basado en las siguientes características técnicas:

Soporte de Runtimes

El SO debe soportar los siguientes runtimes esenciales para la aplicación:

  • OpenJDK 11/17: Para la ejecución del backend Spring Boot
  • Node.js LTS: Para la ejecución del frontend React
  • MySQL 8: Como sistema de gestión de base de datos
  • Maven 3.6+: Para la construcción del proyecto Java

Seguridad

Características de seguridad implementadas:

  • Usuarios dedicados no-root para cada servicio
  • Firewall configurado con puertos mínimos necesarios
  • Gestión de secretos mediante variables de entorno
  • Configuración de AppArmor/SELinux para contención de procesos
  • Actualizaciones automáticas de seguridad habilitadas

Disponibilidad y Mantenimiento

Características para garantizar alta disponibilidad:

  • Soporte para backups automáticos y restauraciones
  • Monitorización con Prometheus + Grafana
  • Sistema de alertas por consumo de recursos
  • Soporte para virtualización y contenedores
  • Alta compatibilidad con hardware estándar del mercado
Comandos de verificación QA:
java -version # Verificar versión de Java
systemctl status erp-backend.service # Verificar servicio
npm audit # Revisar vulnerabilidades frontend

2 • Organizaciones de Estándares

Organizaciones de Estandarización en Redes y Networking

Las siguientes organizaciones son responsables de desarrollar y mantener los estándares internacionales en el área de redes y telecomunicaciones:

Principales Organismos de Estandarización

  • ISO (International Organization for Standardization): Define modelos de referencia como OSI para interoperabilidad entre sistemas.
  • IEEE (Institute of Electrical and Electronics Engineers): Desarrolla estándares para componentes físicos como Ethernet (802.3) y Wi-Fi (802.11).
  • ITU (International Telecommunication Union): Establece estándares globales de telecomunicaciones y gestión del espectro radioeléctrico.
  • IETF (Internet Engineering Task Force): Desarrolla y promueve estándares de Internet a través de RFCs (TCP, IP, HTTP, TLS).
  • W3C (World Wide Web Consortium): Desarrolla estándares para la web (HTML, CSS, accesibilidad).
  • ANSI (American National Standards Institute) y ETSI (European Telecommunications Standards Institute): Organismos regionales que armonizan estándares a nivel nacional y continental.

Nota: El modelo OSI, desarrollado por ISO, proporciona un marco de referencia para la comprensión de las comunicaciones en red a través de sus 7 capas, facilitando la interoperabilidad entre sistemas de diferentes fabricantes.

3 • Familias de Protocolos

Familias de Protocolos de Transmisión y Recepción de Datos

En las comunicaciones de red, existen dos grandes familias de protocolos según su método de operación:

Protocolos Orientados a Conexión (TCP)

Establecen una conexión formal antes de transmitir datos, garantizando entrega confiable y ordenada:

  • Requieren handshake inicial (SYN, SYN-ACK, ACK)
  • Control de flujo y congestión
  • Retransmisión de paquetes perdidos
  • Entrega ordenada de datos
  • Usos comunes: HTTP/HTTPS, SSH, FTP, SMTP, bases de datos

Protocolos No Orientados a Conexión (UDP)

Transmiten datos sin establecer conexión previa, priorizando velocidad sobre confiabilidad:

  • Sin handshake inicial
  • Sin garantías de entrega
  • Sin control de flujo o congestión
  • Menor overhead y latencia
  • Usos comunes: DNS, VoIP, video streaming, juegos online

4 • Medios de Transmisión

Medios de Transmisión Guiados y No Guiados

Los medios de transmisión constituyen el canal físico a través del cual se propagan las señales entre emisor y receptor:

Medios Guiados

Utilizan un conductor físico para guiar las señales:

  • Par Trenzado: Económico, común en redes Ethernet (CAT5e, CAT6, CAT7)
  • Cable Coaxial: Mayor ancho de banda, usado en TV por cable y conexiones antiguas
  • Fibra Óptica: Alta velocidad, larga distancia, inmune a interferencias (FO monomodo/multimodo)

Medios No Guiados

Transmiten señales sin un conductor físico, mediante ondas electromagnéticas:

  • Wi-Fi (IEEE 802.11): Flexibilidad de conexión, susceptible a interferencias
  • Bluetooth: Corto alcance, ideal para dispositivos personales
  • Infrarrojos (IR): Corto alcance, requiere línea de vista
  • Microondas: Largo alcance, usado en enlaces punto a punto
  • Satelital: Cobertura global, alta latencia

Para el ERP_SENA_Final: Se recomienda el uso de fibra óptica para el backbone de red y conexiones cableadas (par trenzado CAT6) para estaciones de trabajo, complementado con Wi-Fi 6 para dispositivos móviles.

5 • Plan de Control de Calidad

Acciones de QA para el Despliegue

Plan de pruebas para garantizar la calidad del despliegue del ERP_SENA_Final:

  1. Pruebas de Instalación: Verificar la correcta instalación de todos los componentes en Ubuntu Server
  2. Pruebas de Integración: Validar la comunicación entre frontend, backend y base de datos
  3. Pruebas de Rendimiento: Evaluar tiempos de respuesta bajo carga con JMeter o k6
  4. Pruebas de Seguridad: Escaneo de vulnerabilidades con OWASP ZAP y revisión de configuración
  5. Pruebas de Recuperación: Verificar procedimientos de backup y restauración
  6. Pruebas de Red: Simular condiciones de red adversas (pérdida de paquetes, latencia)
  7. Pruebas de Compatibilidad: Verificar funcionamiento en diferentes navegadores y dispositivos
Comandos para pruebas QA:
mvn test # Ejecutar pruebas unitarias backend
npm test # Ejecutar pruebas frontend
mysql -u root -p < database_schema.sql # Importar esquema de BD
systemctl enable erp-backend # Habilitar inicio automático
