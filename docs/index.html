1 • Sistema Operativo

Características del Sistema Operativo Seleccionado

Para el despliegue del sistema ERP_SENA_Final, se recomienda utilizar Ubuntu Server LTS como sistema operativo principal, basado en las siguientes características técnicas:

Soporte de Runtimes

El SO debe soportar los siguientes runtimes esenciales para la aplicación:

  • OpenJDK 11/17: Para la ejecución del backend Spring Boot
  • Node.js LTS: Para la ejecución del frontend React
  • MySQL 8: Como sistema de gestión de base de datos
  • Maven 3.6+: Para la construcción del proyecto Java

Seguridad

Características de seguridad implementadas:

  • Usuarios dedicados no-root para cada servicio
  • Firewall configurado con puertos mínimos necesarios
  • Gestión de secretos mediante variables de entorno
  • Configuración de AppArmor/SELinux para contención de procesos
  • Actualizaciones automáticas de seguridad habilitadas

Disponibilidad y Mantenimiento

Características para garantizar alta disponibilidad:

  • Soporte para backups automáticos y restauraciones
  • Monitorización con Prometheus + Grafana
  • Sistema de alertas por consumo de recursos
  • Soporte para virtualización y contenedores
  • Alta compatibilidad con hardware estándar del mercado
Comandos de verificación QA:
java -version # Verificar versión de Java
systemctl status erp-backend.service # Verificar servicio
npm audit # Revisar vulnerabilidades frontend

3 • Proceso de Instalación

Guía Paso a Paso de Instalación

Siga estos pasos para instalar correctamente el ERP_SENA_Final en su sistema:

Preparación del Sistema Operativo

Instalar Ubuntu Server 20.04 LTS o superior con las siguientes especificaciones:

# Actualizar sistema
sudo apt update && sudo apt upgrade -y

# Instalar herramientas básicas
sudo apt install -y curl wget git vim

Instalar Java y Maven

El backend requiere Java 11 o 17 y Maven para su construcción:

# Instalar OpenJDK 17
sudo apt install -y openjdk-17-jdk

# Instalar Maven
sudo apt install -y maven

# Verificar instalación
java -version
mvn -version

Instalar Node.js y NPM

El frontend React requiere Node.js LTS:

# Instalar Node.js LTS
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt install -y nodejs

# Verificar instalación
node --version
npm --version

Instalar y Configurar MySQL

Instalar MySQL Server y crear la base de datos:

# Instalar MySQL Server
sudo apt install -y mysql-server

# Configurar seguridad
sudo mysql_secure_installation

# Crear base de datos y usuario
sudo mysql -e "CREATE DATABASE erp_db;"
sudo mysql -e "CREATE USER 'erp_user'@'localhost' IDENTIFIED BY 'password';"
sudo mysql -e "GRANT ALL PRIVILEGES ON erp_db.* TO 'erp_user'@'localhost';"
sudo mysql -e "FLUSH PRIVILEGES;"

Desplegar la Aplicación

Clonar el repositorio y desplegar la aplicación:

# Clonar repositorio
git clone https://github.com/fergmoon1/ERP_SENA_Final.git
cd ERP_SENA_Final

# Compilar y ejecutar backend
cd backend
mvn clean package
java -jar target/erp-backend.jar

# Instalar y ejecutar frontend (en otra terminal)
cd frontend
npm install
npm start
4 • Licenciamiento

Tipos de Licencia de Software

Es importante considerar los aspectos de licenciamiento para el software utilizado:

Software Libre (Ubuntu, MySQL Community)

Distribuido bajo licencias de código abierto que permiten:

  • Uso sin restricciones
  • Modificación del código fuente
  • Distribución libre
  • Sin costos de licencia

Software Propietario (Windows Server)

Distribuido bajo licencias restrictivas que:

  • Requieren pago de licencias
  • Limitan modificación y distribución
  • Incluyen soporte oficial
  • Tienen restricciones de uso

Para el ERP_SENA_Final: Se recomienda utilizar software de código abierto (Ubuntu, MySQL Community) para reducir costos de licenciamiento y tener mayor control sobre el entorno.

5 • Control de Calidad

Verificación de la Instalación

Procedimientos para verificar que la instalación se realizó correctamente:

  1. Pruebas de Instalación: Verificar la correcta instalación de todos los componentes
  2. Pruebas de Integración: Validar la comunicación entre frontend, backend y base de datos
  3. Pruebas de Rendimiento: Evaluar tiempos de respuesta bajo carga
  4. Pruebas de Seguridad: Verificar configuración de firewall y usuarios
  5. Pruebas de Recuperación: Validar procedimientos de backup y restauración
Comandos para verificación:
systemctl status mysql # Verificar estado de MySQL
curl http://localhost:8080/api/health # Verificar API
npm test # Ejecutar pruebas frontend
mvn test # Ejecutar pruebas backend
