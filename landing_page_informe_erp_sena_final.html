1 • Resumen del repositorio

Visión general y elementos técnicos

El repositorio ERP_SENA_Final contiene una solución ERP ligera para gestión de inventario, clientes, pedidos y facturación. Para un equipo QA/DevOps, es crucial comprender su arquitectura antes del despliegue y validación.

A
Back-end (Java / Spring Boot)

Aplicación construida sobre Spring Boot (pom.xml). QA debe verificar: versión de Java (11/17), controladores REST, seguridad, y configuración de perfiles (dev/prod). Revisión de manejo de excepciones y logs.

B
Base de datos (MySQL)

Incluye script SQL para crear tablas. QA revisa integridad referencial, índices y migraciones. Se recomienda probar backups y restauraciones en staging.

C
Front-end (React)

Frontend con Node.js. QA ejecuta npm audit, revisa dependencias, realiza pruebas de interfaz y verifica CORS y tiempos de respuesta de API.

2 • Características del Sistema Operativo

Requisitos del SO — Ubuntu Server LTS / Debian

Linux es la opción recomendada para producción por estabilidad, soporte a runtimes (Java, Node, MySQL), comunidad y seguridad.

Soporte de Runtimes

  • OpenJDK 11/17: validar con java -version.
  • Maven 3.6+: construir backend con mvn test.
  • Node.js LTS: ejecutar npm install y npm start.
  • MySQL 8: validar con clientes y pruebas de integridad.

Seguridad

  • Usuarios dedicados no-root para cada servicio.
  • Firewall con puertos mínimos abiertos.
  • Gestión de secretos con variables de entorno o vault.
  • AppArmor/SELinux configurado.

Disponibilidad

Implementar backups automáticos, monitorización (Prometheus+Grafana) y alertas de recursos.

Ejemplo QA:
systemctl status erp-backend.service
3 • Organizaciones de estándares

Entidades clave en redes y telecomunicaciones

  • ISO: define modelos de referencia como OSI.
  • IEEE: estándares físicos (Ethernet, Wi-Fi).
  • ITU: telecomunicaciones globales, espectro.
  • IETF: RFCs para TCP, UDP, HTTP, TLS.
  • W3C: estándares web (HTML, accesibilidad).
  • ANSI/ETSI: armonización regional.
4 • Familias de protocolos

Comparativa y usos

Orientados a conexión (TCP)

  • Entrega ordenada, retransmisiones y control de flujo.
  • Usos: HTTP/HTTPS, bases de datos, transacciones.

No orientados a conexión (UDP)

  • Sin garantías, baja latencia.
  • Usos: DNS, streaming, VoIP.
5 • Medios de transmisión

Guiados y no guiados

Guiados

  • Par trenzado: económico y común en LAN.
  • Cable coaxial: uso limitado en redes modernas.
  • Fibra óptica: altas velocidades y largas distancias.

No guiados

  • Wi-Fi: flexible pero sensible a interferencias.
  • Bluetooth/IR: corto alcance, IoT.
  • Microondas/satélite: enlaces remotos, latencia alta.
6 • Acciones QA

Plan de pruebas

  1. Pruebas end-to-end: flujo completo de pedidos y facturación.
  2. Smoke tests: ejecución de mvn test y npm start.
  3. Pruebas de carga con JMeter o k6.
  4. Pruebas de seguridad: OWASP, revisión de secrets.
  5. Pruebas de recuperación: restaurar backups.
  6. Pruebas de red: simular latencia y pérdida.
  7. Pruebas de compatibilidad: navegadores y móviles.
